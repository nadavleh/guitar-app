FretPal — GUI Redesign Mockups

Goal: kill the scrolling. Today every tool opens a tall modal sheet of stacked sliders & chip-rows, and the app is locked to landscape. These mockups make controls fit on one screen via a fixed control dock + persistent navigation, and they work in both portrait and landscape.

● root   ● chord tone   ● scale tone Dock = persistent control strip (no modal, no page-scroll) Rail / Tab bar = always-visible navigation

Two concepts below. Concept A = "Studio" (nav rail + bottom dock). Concept B = "Deck" (top tabs + responsive cards). Each shown in landscape & portrait. Pick one (or mix) and I'll build it.

Concept A — “Studio” persistent nav rail (landscape) / bottom tab bar (portrait) + a fixed control dock under the neck

The neck always stays visible; the dock under it changes with the active tool. Controls are laid out as compact label→chips rows and segmented toggles that fit in ~3 rows — no scrolling. Quick audio knob (🎚) lives in the top bar everywhere.
